# LP構成 壁打ちメモ

## 前提

既存の `index.html` / `style.css` / `script.js` は参考用の提案資料であり、今回作るQR遷移先LP本体とは切り離して考える。

今回のLPは、投票済証明書のQRを読み取った人がスマホで開くページ。

目的は、投票後のユーザーを以下の流れに乗せること。

```text
QRでサイトを開く
→ ファーストビュー
→ 対象店舗・商品を見る
→ 使い方とQ&Aで不安解消
→ 近くのお店を地図で開く
→ 来店
```

## 推奨構成

1. ファーストビュー
2. 対象店舗・商品
3. 使い方
4. Q&A
5. 地図CTA
6. 来店CTA

## ファーストビュー

役割:

- 投票後に開いた人へ「何のページか」を即時に伝える
- 次に押すボタンを明確にする
- 地域回遊の雰囲気を出す

必要要素:

- メインコピー
- サブコピー
- CTA `特典を見る！`
- サブCTA `地図で近くのお店を探す`
- 小さな安心文言

コピー案:

```text
投票おつかれさまでした。
帰り道に寄れる、稲毛の地域応援スポットを見つけよう。
```

## 対象店舗・商品

役割:

- ユーザーが最も知りたい「何が使えるか」を見せる
- 店舗単位ではなく、特典単位で判断できるようにする

カードに入れる情報:

- 特典名
- 店舗名
- 対象商品・サービス
- 条件
- 徒歩目安
- 地図リンク

考え方:

- 見出しは「対象店舗」より「使える特典」のほうが行動に近い
- 実店舗未確定の場合はサンプル表記を強く出す

## 使い方

3ステップ案:

1. 投票済証明書を受け取る
2. LPで対象店舗・商品を確認する
3. 店舗で証明書を提示する

## Q&A

入れる質問:

- 投票済証明書を忘れた場合は？
- 何回使える？
- どの店舗で使える？
- 有効期限は？
- 店舗で何を見せる？
- QRが読み取れない場合は？

## 地図CTA

仕様:

- 地図は別タブで開く
- `target="_blank"` と `rel="noopener noreferrer"` を付ける
- ファーストビュー、特典カード、Q&A後に置く

## 指摘

LPは企画説明より行動導線を優先する。QRから来た人は、企画の背景よりも「どこで何が使えるか」を先に知りたい。

「対象店舗」と「商品」は分けすぎない。ユーザーは店舗名よりも、先に「自分に使えそうか」を判断する。

カードは以下の順がよい。

```text
特典内容
→ 対象商品
→ 店舗名
→ 条件
→ 地図
```

割引を主役にしすぎない。主語は「地域応援」「まち歩き」「投票後の寄り道」に寄せる。

避けたい表現:

- 投票したら得をする
- 投票の対価
- 投票すれば割引
