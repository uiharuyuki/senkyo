# いなげ若者選挙プロジェクト LP

投票済証明書のQRから開く、稲毛の地域応援スポット案内LPです。

メインコンセプト:

> 投票した帰りに、稲毛を少し歩きたくなる。

## 現在の構成

50店舗規模まで増えても更新しやすいように、店舗情報はHTMLへ直書きせず `stores.json` で管理します。

```text
index.html    QR着地点兼 対象店舗一覧。ファストビュー画像、コンパクトなカード型検索・ジャンル絞り込み、Q&Aを掲載。
how-to-use.html  使い方ページ。ファストビュー画像と利用手順を掲載。
shop.html     店舗詳細テンプレート。URLの ?id=... から店舗と商品画像を表示。
stores.json   店舗・対象商品・条件・Googleマップ検索文言のデータ。
app.js        店舗一覧と店舗詳細の描画。
style.css     共通スタイル。スマホfirst。生成イラストと商品写真風カード画像を使用。
assets/       ファストビュー・図解用PNG、商品写真風カード画像、ジャンル・店舗・商品用SVG。
```

旧提案資料として使っていた `index.md`、旧 `script.js`、旧ビジュアル画像、仮の `lp.html` / `lp.css`、旧互換用の `shops.html` は削除済みです。トップページは対象店舗一覧から始めます。

## ユーザーフロー

1. 投票済証明書のQRから `index.html` を開く
2. `index.html` で対象店舗カードを見る
3. 必要に応じて一覧下のQ&A、または `how-to-use.html` で使い方を確認する
4. `shop.html?id=...` で対象商品・特典・利用条件を見る
5. 店舗カードまたは詳細ページからGoogleマップ検索を別タブで開く
6. 店舗へ来店し、必要に応じて投票済証明書を提示する

## 設計方針

- スマホfirst
- トップページは対象店舗一覧から始める
- 地図埋め込みは置かない
- Googleマップは店舗カード・詳細ページのボタンから別タブで開く
- 店舗一覧はカード型
- 店舗カードは一覧性を優先し、対象商品・特典・利用条件は詳細ページで表示する
- Q&Aは対象店舗ページの特典一覧下、「もっと見る」ボタンの下に置く
- 割引そのものを主役にしすぎず、地域応援・まち歩きを主語にする

## 店舗追加方法

`stores.json` に店舗オブジェクトを追加します。

```json
{
  "id": "sample-shop",
  "name": "店舗名",
  "sample": true,
  "genre": "カフェ",
  "area": "稲毛駅周辺",
  "distance": "稲毛駅から徒歩5分",
  "catch": "詳細ページなどで使う短い紹介文",
  "summary": "詳細説明",
  "mapQuery": "Googleマップで検索したい文言",
  "tags": ["カフェ", "地域応援"],
  "conditions": "利用条件",
  "products": [
    {
      "name": "対象商品名",
      "benefit": "特典内容",
      "condition": "商品ごとの条件"
    }
  ]
}
```

`mapQuery` は `app.js` 側でGoogleマップ検索URLに変換します。

詳細ページで商品やメニューの雰囲気を見せたい場合は、生成した商品写真風画像を `assets/` に置き、店舗オブジェクトに `cardImage` を追加します。実在店舗の写真に見せないため、ロゴ、読める文字、人物、実在ブランドは入れません。
