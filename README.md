# いなげ若者選挙プロジェクト LP

投票済証明書のQRから開く、稲毛の地域応援スポット案内LPです。

メインコンセプト:

> 投票した帰りに、稲毛を少し歩きたくなる。

## 現在の構成

50店舗規模まで増えても更新しやすいように、店舗情報はHTMLへ直書きせず `stores.json` で管理します。

```text
index.html    QR着地点。FVと「特典を見る！」、サイト説明、使い方。
shops.html    対象店舗一覧。カード型で検索・ジャンル絞り込み。
shop.html     店舗詳細テンプレート。URLの ?id=... から店舗を表示。
faq.html      Q&Aページ。必要に応じて拡張。
stores.json   店舗・対象商品・条件・Googleマップ検索文言のデータ。
app.js        店舗一覧と店舗詳細の描画。
style.css     共通スタイル。スマホfirst。画像なしの色面デザイン。
```

旧提案資料として使っていた `index.md`、旧 `script.js`、旧ビジュアル画像、仮の `lp.html` / `lp.css` は削除済みです。FVは画像なしで構成します。

## ユーザーフロー

1. 投票済証明書のQRから `index.html` を開く
2. ファーストビューで `特典を見る！` を押す
3. `shops.html` で対象店舗カードを見る
4. 店舗カードから `詳細を見る` を押す
5. `shop.html?id=...` で対象商品と条件を見る
6. 店舗カードまたは詳細ページからGoogleマップ検索を別タブで開く
7. 店舗へ来店し、必要に応じて投票済証明書を提示する

## 設計方針

- スマホfirst
- FVの主要CTAは `特典を見る！` のみにする
- 地図埋め込みは置かない
- Googleマップは店舗カード・詳細ページのボタンから別タブで開く
- 店舗一覧はカード型
- 店舗詳細で対象商品を一覧表示する
- Q&Aは独立ページとして拡張できるようにする
- 割引そのものを主役にしすぎず、地域応援・まち歩きを主語にする

## 店舗追加方法

`stores.json` に店舗オブジェクトを追加します。

```json
{
  "id": "sample-shop",
  "name": "店舗名",
  "sample": true,
  "genre": "カフェ",
  "area": "稲毛駅周辺",
  "distance": "稲毛駅から徒歩5分",
  "catch": "店舗カードに出す短い紹介文",
  "summary": "詳細説明",
  "mapQuery": "Googleマップで検索したい文言",
  "tags": ["カフェ", "地域応援"],
  "conditions": "利用条件",
  "products": [
    {
      "name": "対象商品名",
      "benefit": "特典内容",
      "condition": "商品ごとの条件"
    }
  ]
}
```

`mapQuery` は `app.js` 側でGoogleマップ検索URLに変換します。

## 注意

現時点の店舗・商品・特典はサンプルです。実施時は、参加店舗、対象商品、利用条件、有効期限、投票済証明書の扱いを関係者確認のうえ掲載します。
