# いなげ若者選挙プロジェクト LP制作メモ

## 目的

投票済証明書のQRから開く、稲毛の地域回遊LPを制作する。

このLPは「投票した人に割引を見せるページ」ではなく、投票後に稲毛の街を少し歩き、地域のお店や商品と出会うための入口として設計する。

メインコンセプト:

> 投票した帰りに、稲毛を少し歩きたくなる。

## 想定するユーザーの流れ

1. 投票済証明書のQRを読み取る
2. LPのファーストビューを見る
3. 「特典を見る」から対象店舗・商品を見る
4. 使い方とQ&Aで不安を解消する
5. 「近くのお店で使おう」から地図を別タブで開く
6. 店舗へ来店する
7. 必要に応じて投票済証明書を提示して特典を利用する

## 必要なLP要素

### 1. ファーストビュー

QRから来た直後に、何のページか、何ができるか、次に何を押せばよいかを一瞬で伝える。

入れる要素:

- メインコピー
- 短い説明文
- メインCTA「特典を見る」
- サブCTA「地図で探す」
- 投票済証明書から来た人向けの安心文言
- 稲毛の街歩き感が出るビジュアル

コピー案:

> 投票おつかれさまでした。  
> 帰り道に寄れる、稲毛の地域応援スポットを見つけよう。

CTA案:

- 特典を見る
- 地図で近くのお店を探す

補足文言案:

> 掲載情報・特典内容は参加店舗ごとに異なります。利用前に各店舗の条件をご確認ください。

### 2. 対象店舗・商品

ユーザーが一番見たい部分。店舗名だけでなく「何が使えるのか」を先に見せる。

カードに入れる要素:

- 店舗名
- 商品・サービス名
- ジャンル
- 特典内容
- 徒歩目安
- 証明書提示が必要か
- 地図リンク

見せ方の提案:

- 「店舗」より先に「特典内容」が目に入るカードにする
- 「カフェ」「ベーカリー」「本屋」などジャンルで絞れるUIを入れる
- 実店舗が未確定なら「サンプル店舗」と明記する

### 3. 使い方

初見でも迷わないように3ステップで説明する。

例:

1. 投票済証明書を受け取る
2. LPで対象店舗・商品を確認する
3. 店舗で証明書を提示する

注意:

- 実施前に、投票済証明書の扱い、特典提供、告知表現について選挙管理委員会・関係者確認を行う
- 「投票の対価」と誤解される表現は避け、「地域回遊」「地域応援」を主語にする

### 4. Q&A

不安解消のため、CTA前後に置くと効果が高い。

入れる質問案:

- 投票済証明書を忘れた場合は使えますか？
- 何回でも使えますか？
- どの店舗で使えますか？
- 特典内容はいつまで有効ですか？
- 店舗で何を見せればよいですか？
- QRが読み取れない場合はどうすればよいですか？

### 5. 地図リンク

地図は別タブで開く。LPから離れても戻りやすいようにする。

実装方針:

```html
<a href="https://maps.google.com/..." target="_blank" rel="noopener noreferrer">
  地図で近くのお店を探す
</a>
```

地図リンクは以下に置く:

- ファーストビューのサブCTA
- 店舗カード内
- 使い方・Q&A後のCTA

### 6. CTA

メインCTAは「特典を見る！」に統一する。

配置:

- ファーストビュー
- 対象店舗・商品の直前
- Q&Aの後
- スマホ画面下部の固定CTA

## 指摘

### 既存ページは「提案資料」であり、QR遷移先LPではない

現在の `index.html` / `style.css` / `script.js` は参考用の提案資料として扱う。QRから来た一般ユーザー向けのLP本体は、既存ファイルを直接作り替えず、新規ファイルとして分ける。

LPでは、企画説明よりも「今すぐ何ができるか」を優先する。

改善:

- 表紙・2案比較・企画説明を減らす
- 対象店舗・商品・地図・使い方を前に出す
- 「提案用デモ」表記は開発段階のみ残す

### ファーストビューに具体的な行動が必要

「稲毛を歩きたくなる」だけでは次の行動が弱い。

改善:

- 「特典を見る」を最も目立たせる
- 「地図で探す」はサブ導線にする
- 「証明書を持っている人が使える」ことを短く伝える

### 商品と店舗の関係を整理する必要がある

「対象店舗」と「商品」が混ざるとユーザーが迷う。

改善:

- カードの主語を「特典」にする
- 店舗名、対象商品、条件を同じカード内にまとめる
- 店舗一覧ではなく「使える特典一覧」として見せる

### Q&Aは後半だけでなく、利用直前にも必要

不安が残ると来店につながらない。

改善:

- 店舗カードの下に「使い方3ステップ」を置く
- Q&Aの後に地図CTAを置く
- 証明書提示、有効期限、対象条件を明確にする

## 推奨LP構成

1. ファーストビュー
2. 使える特典一覧
3. ジャンル別・店舗別の対象商品
4. 使い方3ステップ
5. Q&A
6. 地図で探すCTA
7. 来店を後押しする締めコピー

## 実装方針

このリポジトリは静的HTML / CSS / JavaScriptで進める。

既存の `index.html` / `style.css` / `script.js` は参考用。新規LPのたたき台は `lp.html` / `lp.css` として独立させる。

優先して行うこと:

- `lp.html` をQR遷移先LPの構成確認用として作る
- `lp.css` をLP用に整理する
- 必要になるまでJavaScriptは追加しない
- スマホ前提で設計する
- 外部地図リンクは別タブにする
- 実店舗未確定の場合はサンプル表記を入れる

GitHub管理先:

```text
https://github.com/uiharuyuki/senkyo
```

ローカルでまだGit管理されていない場合は、作業前に `git init` と `git remote add origin https://github.com/uiharuyuki/senkyo.git` を行う。
